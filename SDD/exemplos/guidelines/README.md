# projetos já em andamento.
- Se seu projeto já estiver criado, não se preocupe em criar guidelines. 
- Apenas execute o /guidelines que ele criará os arquivos com base no projeto atual.
- Será identificada toda a stack de desenvolvimento e padrões adotados e criados os guidelines.

# projetos novos
- Um arquiteto de software e/ou um desenvolvedor experiente deve definir os guidelines do projeto
- Não há um padrão de nomenclatura, mas é necessário estar no formato md para facilitar o entendimento.
- Utilize padrões já existentes, tanto de mercado quando do próprio projeto.
- Documentação em PDF, txt, csv etc podem ser utilizadas como referência, mas exigem referenciamento direto no README.md da raiz do projeto (veja o README.md do projeto SDD).
